条件语句

WHERE语句

​ 用来描述哪些行应该进入结果，具体就是通过 condition条件 限定这些行的属性满足某些具体条件。

Operator(关键字) Condition(解释) SQL Example(例子)
= , != , < , <= , > , >= 基础的 大于，等于等比较 col_name != 4
BETWEEN…AND… 在两个数之间 col_name BETWEEN 1.5 AND 10.5
NOT BETWEEN … AND … 不在两个数之间 col_name NOT BETWEEN 1 AND 10
IN (…) 在一个列表 col_name IN (2, 4, 6)
NOT IN (…) 不在一个列表 col_name NOT IN (1, 3, 5)
Operator(关键字) Condition(解释) SQL Example(例子)
= 完全等于 col_name = “abc”
!= or <> 不等于 col_name != “abcd”
% 通配符，代表匹配0个以上的字符 col_name LIKE “%AT%”
“%AT%” 代表AT 前后可以有任意字符
_ 和% 相似，代表1个字符 col_name LIKE “AN_”
LIKE （模糊查询）没有用通配符等价于 = col_name LIKE “ABC”
NOT LIKE 没有用通配符等价于 != col_name NOT LIKE “ABCD”
IN (…) 在列表 col_name IN (“A”, “B”, “C”)
NOT IN (…) 不在列表 col_name NOT IN (“D”, “E”, “F”)

查询结果过滤

DISTINCT语句

​ DISTINCT 语法介绍，我们拿之前的 Movies表来说，可能很多电影都是同一年Year发布的，如果你想要按年份排重，一年只能出现一部电影到结果中， 你可以用 DISTINCT 关键字来指定某个或某些属性列唯一返回。写作：DISTINCT Year

选取出唯一的结果的语法

查询结果排序

ORDER BY语句

​ 我们可以用 ORDER BY col_name 排序的语法来让结果按一个或多个属性列做排序.

​ ORDER BY col_name 这句话的意思就是让结果按照 col_name 列的具体值做 ASC升序 或 DESC 降序，对数字来说就是升序 1，2，3，… 或降序 … 3,2,1 . 对于文本列，升序和降序指的是按文本的字母序。

选取部分结果

LIMIT 和 OFFSET

​ LIMIT 和 OFFSET 子句通常和ORDER BY 语句一起使用，当我们对整个结果集排序之后，我们可以 LIMIT来指定只返回多少行结果 ,用 OFFSET来指定从哪一行开始返回。你可以想象一下从一条长绳子剪下一小段的过程，我们通过 OFFSET 指定从哪里开始剪，用 LIMIT 指定剪下多少长度。
